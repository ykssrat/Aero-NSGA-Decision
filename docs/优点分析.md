# OptiSelect — 工程优点分析

---

## 一、算法层面

### 1.1 全局搜索能力强，避免局部最优陷阱

NSGA-II 作为进化类多目标优化算法，通过种群级别的并行搜索（本工程设定种群规模 100、迭代 100 代），能够在高维离散组合空间中有效探索全局，克服了传统贪心策略或穷举搜索容易陷入局部最优的根本性问题。

### 1.2 真正的多目标优化，而非简单加权求和

系统并未将"成本"和"性能"简单相加为一个标量指标，而是在算法内部保持两个目标的独立性，通过**非支配排序**（`NonDominatedSorting.m`）和**拥挤度距离**（`CalcCrowdingDistance.m`）机制提取出完整的 **Pareto 前沿**，让决策者看到所有不可相互替代的最优折中方案，而非仅仅一个"唯一解"。

### 1.3 科学的最终决策——TOPSIS 接棒

在获得 Pareto 前沿后，系统使用 TOPSIS（`TOPSIS_Selection.m`）进行加权综合决策。TOPSIS 的核心思想是"距离理想解最近且距离负理想解最远"，这比简单取 Pareto 前沿中某个端点更加科学合理。用户可以通过调节成本/性能权重比（当前为 0.3:0.7）来体现自身的偏好倾向。

## 二、工程设计层面

### 2.1 约束预筛选大幅减少无效搜索

在进入遗传算法迭代之前，系统先根据用户输入的**三类布尔属性约束**（属性3：如抗电磁干扰；属性4：国产化；属性5：飞行经历）对元器件数据库进行预筛选。这意味着：
- 不符合硬性要求的型号在优化开始前就被排除
- 算法的搜索空间仅限于合规候选，避免了大量无效评估
- 搜索效率和结果质量均显著提升

### 2.2 模块化程度高，职责划分清晰

全部 15 个 `.m` 文件按功能严格分离：
| 功能类别 | 脚本数量 | 示例 |
|---------|---------|------|
| 遗传算子 | 3 个 | `IntegerCrossover_Filtered.m`、`IntegerMutate_Filtered.m`、`Dominates.m` |
| 排序与选择 | 3 个 | `NonDominatedSorting.m`、`CalcCrowdingDistance.m`、`SortPopulation.m` |
| 目标函数与评价 | 3 个 | `PC_CostFunction_Filtered.m`、`CalculateCompatibility_WithCase.m`、`CalculateDomesticBoost.m` |
| 结果输出 | 3 个 | `DisplayPCConfiguration_Filtered.m`、`PlotFinalResults.m`、`PlotPCFront.m` |
| 数据保存 | 1 个 | `SaveResultsToFile_Filtered.m` |
| 决策分析 | 1 个 | `TOPSIS_Selection.m` |
| 主控流程 | 1 个 | `computer.m` |

这种设计使得任何单一模块的修改（例如更换交叉算子的策略或更换评价函数的加权逻辑）都不影响其他模块，维护成本低。

### 2.3 灵活的参数7（可选组件）支持

系统内置了对"**第7项参数（可选组件）**"的完整支持。用户在运行时可交互选择是否纳入该组件，系统会自动切换到对应的数据库（`Components_WithCase` 或 `Components_WithoutCase`），并确保整个优化流程（交叉、变异、目标函数计算、结果呈现）中的逻辑一致性。这种设计模式使系统可以灵活适应"有/无某可选模块"的不同业务需求。

### 2.4 国产化性能影响的非线性建模

`CalculateDomesticBoost.m` 模拟了一个现实中常见但难以建模的现象：**当国产化替代组件的数量增加时，整体系统性能会呈非线性递减趋势**（例如1个国产件降低1%、3个降低6%、7个降低28%）。这使得优化结果能较真实地反映"国产化率"与"性能"之间的权衡关系。

## 三、用户交互层面

### 3.1 交互式输入，零代码配置筛选条件

用户完全不需要修改任何代码即可定制筛选和优化行为——通过 MATLAB 命令行的交互式 `input()` 提示，逐项设置每个参数的属性4和属性5需求。这对非编程背景的工程师来说尤为友好。

### 3.2 结果呈现多维度、可追溯

- **控制台输出**：通过 `DisplayPCConfiguration_Filtered.m` 详细打印最佳方案的每个组件型号、成本、性能及属性状态
- **可视化**：`PlotFinalResults.m` 一次性生成 Pareto 前沿散点图、成本分布直方图、性能分布直方图、性价比分析图四个子图
- **文件保存**：`SaveResultsToFile_Filtered.m` 同时输出 `.mat`（可重新加载）和 `.txt`（人类可读）两种格式，确保结果可追溯、可复现
